srsRAN 5G NR jBPF Telemetry Pipeline — Project Summary & Current State
======================================================================

Project Overview
----------------
A complete 5G NR MAC-layer telemetry pipeline using srsRAN Project (with jBPF
hooks), a custom ZMQ channel broker for RF impairment simulation, and a Grafana
+ InfluxDB dashboard for real-time visualization.  The system runs entirely on a
single Ubuntu 22.04 machine using ZMQ virtual radio (no physical RF hardware).

Key capabilities:
  - 11 jBPF codelet sets (~60 codelets), 17 telemetry schemas
  - AWGN + Rician flat fading + CW/narrowband interference injection
  - 3GPP frequency-selective fading (EPA/EVA/ETU) via GRC Python broker
  - Live QT GUI with spectrum/constellation visualization
  - End-to-end UE latency & throughput monitoring (DL/UL iperf3, ping RTT)
  - 20+ stress scenarios for anomalous dataset collection


Architecture
------------
jrtc (runtime controller)
  -> ZMQ Channel Broker (C: Rician fading + AWGN + CW interference)
     OR: GRC Python Broker (EPA/EVA/ETU fading + AWGN + CFO + drops +
         CW/narrowband interference + live QT GUI)
    -> gNB (5G base station, ~60 jBPF eBPF codelets)
      -> Reverse Proxy (IPC->TCP bridge, port 30450)
      -> Decoder (protobuf->JSON, gRPC:20789, UDP:20788)
        -> telemetry_to_influxdb.py (Python ingestor)
          -> InfluxDB 1.6.7 (port 8086, db=srsran_telemetry, 15 measurements)
            -> Grafana 11.5.2 (port 3000, admin/admin, 39-panel dashboard)
  -> srsUE (UE simulator, ports 2000/2001)
    -> iperf3 UL (port 5201, 10 Mbps UDP uplink)
    -> iperf3 DL (port 5202, 5 Mbps UDP downlink, --reverse)
    -> ping RTT monitoring -> /tmp/ping_ue.log

ZMQ broker ports: gNB 4000/4001 <-> broker <-> UE 2000/2001.


Key Files (repository: scripts/, docs/, project_extension/)
---------------------------------------------------------
scripts/zmq_channel_broker.c (~560 lines)
    C broker — Rician/Rayleigh flat fading + AWGN + CW interference.
    CLI: --snr, --fading, --k-factor, --doppler, --rayleigh,
         --interference-type {cw}, --interference-freq Hz, --sir dB

scripts/srsran_channel_broker.py (~1050 lines)
    GRC Python broker (superset of C broker). Adds EPA/EVA/ETU
    frequency-selective fading, CFO, burst drops, time-varying scenarios,
    CW and narrowband (1 PRB) interference, live QT GUI.
    CLI: same as C broker + --grc, --gui, --profile, --cfo, --drop-prob,
         --scenario, --interference-type {cw,narrowband}

scripts/launch_mac_telemetry.sh (~570 lines)
    One-shot pipeline launcher. Starts all 8 components in order.
    Starts DL iperf3 (--reverse, port 5202) and continuous ping RTT
    monitoring alongside UL iperf3.
    Flags: --fading, --k-factor N, --snr N, --grc, --gui, --profile P,
           --cfo N, --drop-prob N, --scenario S,
           --interference-type T, --interference-freq Hz, --sir dB,
           --no-broker, --no-ue, --no-traffic, --no-grafana

scripts/stop_mac_telemetry.sh (~110 lines)
    Pipeline teardown (reverse order, includes ping stop)

scripts/stress_anomaly_collect.sh (~830 lines)
    Stress anomaly dataset collection. Applies 20+ system-level stressors
    (CPU, memory, network, I/O) to a running pipeline and captures labeled
    MAC telemetry per scenario. Per-scenario UL/DL throughput and ping RTT
    measurement via iperf3. Outputs manifest CSV with 14 columns.

scripts/telemetry_to_influxdb.py (616 lines)
    Python ingestor — tails decoder JSON log, parses 16 protobuf schema
    types, writes to InfluxDB via HTTP API. Handles cumulative->delta
    conversion for MAC stats.

scripts/plot_all_telemetry.py (944 lines)
    15-plot generator for all 17 telemetry schemas. Parses decoder log.

docs/ZMQ_CHANNEL_BROKER_DOCS.md
    Full documentation of the channel broker, channel models (AWGN, Rician,
    EPA/EVA/ETU, CW/narrowband interference), measured results, and
    GRC Python broker details.

docs/PROJECT_CONTINUATION.md
    Full project reference — all phases, file inventory, port map, system
    configuration, how-to-run examples, telemetry schemas, and data quirks.

project_extension/
    Anomalous dataset extension docs and figures.
    - STRESS_ANOMALY_EXTENSION.md — technical write-up of the stress injection
      methodology and results
    - STRESS_ANOMALY_SUMMARY.md — plain-language summary of the anomalous dataset work
    - plot_bep_presentation.py — figure generation script
    - figures/ — 5 PNG figures (hook overview, signal space, anomaly heatmap,
      SINR/MCS invariance, category summary)


What Was Built (Phases)
-----------------------
Phase 1-3 (ZMQ Broker): C broker with AWGN, Rayleigh then Rician fading.
  K=3 dB + SNR=28 dB + fd=5 Hz -> 12 HARQ failures, 15+ min stable connection.

Phase 4-5 (Pipeline + Dashboard): launch/stop scripts, Grafana + InfluxDB.
  39 Grafana panels, 15 InfluxDB measurements, Python ingestor.

Phase 6 (HARQ Tuning): Systematic 6-config sweep to find failure-triggering
  params without crashing. Winner: K=3, SNR=28, fd=5.

Phase 7-8 (Plotting + Validation): 15-plot generator, validated all metrics
  against channel model (SINR 30.3 dB mean, 4.6 dB std, 9.9% deep fades).

Phase 9 (GRC Python Broker): EPA/EVA/ETU FIR fading, CFO, burst drops,
  time-varying scenarios, live QT GUI. Fixed AWGN power bug (unit-power
  assumption vs actual ~0.002 srsRAN sample power).

Phase 10 (Stability Fix): Diagnosed pipeline crashes as iperf3 TUN race
  condition. Fixed: launch script waits up to 15s for TUN IP.

Phase 11 (Stress Scenarios): stress_anomaly_collect.sh with 20+ stressor
  scenarios across CPU/memory/network categories. project_extension/ figures.

Phase 12 (UE Latency & Throughput): DL iperf3 port 5202 --reverse, continuous
  ping RTT monitoring, measure_scenario() in stress script, manifest CSV
  extended with ul_mbps/dl_mbps/rtt_avg_ms/rtt_max_ms columns.

Phase 13 (Interference): CW interference in C broker (interference_apply()),
  CW + narrowband in GRC broker (apply_interference()). Launch flags:
  --interference-type, --interference-freq, --sir. narrowband auto-routes to
  GRC broker. GUI rows 3-4 (SIR slider, type combo, freq slider).
  Validated: SIR=10 -> MAC DL SINR 3.6 dB, ping RTT 39s.


Quick Start
-----------
# C broker, Rician fading (recommended — triggers HARQ failures)
cd ~/Desktop && bash launch_mac_telemetry.sh --fading

# GRC broker, EPA frequency-selective fading
bash launch_mac_telemetry.sh --grc --profile epa --snr 28

# GRC broker with GUI
bash launch_mac_telemetry.sh --gui --fading

# CW interference at 1 MHz, SIR=10 dB
bash launch_mac_telemetry.sh --interference-type cw --sir 10

# Narrowband interference (auto-selects GRC broker)
bash launch_mac_telemetry.sh --interference-type narrowband --sir 15

# Stop everything
bash stop_mac_telemetry.sh

# Collect stress anomaly dataset
bash stress_anomaly_collect.sh --duration 180

# Rebuild C broker
gcc -O2 -o zmq_channel_broker zmq_channel_broker.c -lzmq -lm -lpthread

# Dashboard
http://localhost:3000  (admin/admin)
